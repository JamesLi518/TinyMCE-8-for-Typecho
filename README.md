# TinyMCE-8-for-Typecho

一款专为 Typecho 1.3.0 深度定制的 TinyMCE 8.4.0 富文本编辑器插件。它完美替换了原生的 Markdown 编辑器，并深度集成了 Typecho 的附件管理系统，提供专业、顺滑的写作体验。

✨ 功能亮点
最新核心：基于 TinyMCE 8.4.0 ，享受最现代的富文本编辑体验。

本地/CDN 自由切换：支持完全本地化部署（服务器带宽充足）或使用公共 CDN（减少服务器带宽压力）。

附件管理深度集成：接管原生“附件”面板，点击上传后的图片或文件可直接按 HTML 格式精准插入光标处。

多功能设置面板：

高度自定义：自由设定编辑器高度。

外观皮肤：支持一键切换“明亮模式”与“暗黑模式”。

菜单栏开关：可选择隐藏顶部冗余菜单，打造极简写作环境。

工具栏预设：提供“极简”、“标准”、“全功能”三种模式，满足不同写作习惯。

同步机制：实时自动同步内容至 Typecho 数据库，确保草稿保存与文章发布万无一失。

防止冗余：自动禁用图片粘贴为 Base64 功能，强制引导使用附件系统，保护数据库不被撑爆。

📂 目录结构
请确保您的插件目录结构如下：

plugins/TinyMCE8/

├── tinymce/               # [可选] TinyMCE 8 核心程序文件夹 (本地模式必需)

├── langs/

│   └── zh_CN.js           # 中文语言包

├── Plugin.php             # 插件核心逻辑

└── README.md              # 本说明文件

🚀 安装步骤
下载最新版本文件。

上传文件进行解压。

激活插件：登录 Typecho 后台，进入 “控制台” -> “插件”，找到 TinyMCE8 并点击 “启用”。

自定义配置：点击插件旁边的 “设置”，根据您的需求调整高度、加载方式及皮肤。

⚠️ 重要说明（必看）
1. 关闭 Markdown 解析
由于本插件生成的是原生 HTML 代码，而 Typecho 默认会尝试用 Markdown 引擎解析文章，这可能导致排版错乱。

操作：在Typecho个人设置中找到撰写设置在使用 Markdown 语法编辑和解析内容切换为 “关闭”。

2. 浏览器缓存
如果您在修改了插件设置（如切换皮肤或工具栏）后没看到变化，请按下 Ctrl + F5 (Windows) 或 Cmd + Shift + R (Mac) 强制刷新浏览器缓存。、
